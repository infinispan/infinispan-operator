= Configuring service accounts

[role="_abstract"]
Specify the service account that {brandname} pods use instead of the default service account in the namespace.
This is useful when you need pods to use a specific service account for pulling images from private container registries or for fine-grained RBAC control.

.Prerequisites
* Have a service account created in the same namespace as the `Infinispan` CR.

.Procedure
. Configure the service account name in the `Infinispan` CR.

+
The `spec.serviceAccountName` field sets the service account for {brandname} server pods, GossipRouter pods, and Batch, Backup, and Restore pods.
+
The `spec.configListener.serviceAccountName` field sets the service account for the ConfigListener pod.
The ConfigListener requires specific RBAC permissions to manage Cache CRs, so a separate service account is supported.
+
. If you specify `spec.configListener.serviceAccountName`, you must create the required RBAC resources for the ConfigListener yourself.

. Apply your `Infinispan` CR.
+
If your cluster is running, {ispn_operator} restarts the {brandname} pods in order for the changes to take effect.

NOTE: When `spec.configListener.serviceAccountName` is not specified, {ispn_operator} automatically creates and manages the ServiceAccount, Role, and RoleBinding for the ConfigListener.
