= Creating {brandname} clusters

[role="_abstract"]
Use the `{kube_client}` to provide {ispn_operator} with custom `Infinispan` CR definitions for {brandname} clusters.

.Prerequisites

* Install {ispn_operator}.
//Community
ifdef::community[]
* Have an `oc` or a `kubectl` client.
endif::community[]
//Downstream
ifdef::downstream[]
* Have an `oc` client.
endif::downstream[]

.Procedure

. Create a YAML file that defines your `Infinispan` CR.
+
For example, create a `cr_minimal.yaml` file that specifies the number of {brandname} pods as follows:
+
[source,options="nowrap",subs=attributes+]
----
include::cmd_examples/cat_cr_minimal.adoc[]
----
+
. Apply your `Infinispan` CR.
+
[source,options="nowrap",subs=attributes+]
----
$ {oc_apply_cr} cr_minimal.yaml
----
+
. Watch {ispn_operator} create the {brandname} pods.
+
[source,options="nowrap",subs=attributes+]
----
include::cmd_examples/oc_get_pods_w.adoc[]
----

.Next Steps

Modify the value of the `replicas` field in your `Infinispan` CR, apply the changes, and watch {ispn_operator} scale the cluster up or down.
