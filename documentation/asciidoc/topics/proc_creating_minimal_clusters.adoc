= Creating {brandname} clusters

[role="_abstract"]
Use {ispn_operator} to create clusters of two or more {brandname} pods.

.Prerequisites

* Install {ispn_operator}.
//Community
ifdef::community[]
* Have an `oc` or a `kubectl` client.
endif::community[]
//Downstream
ifdef::downstream[]
* Have an `oc` client.
endif::downstream[]

.Procedure

. Specify the number of {brandname} pods in the cluster with `spec.replicas`
in your `Infinispan` CR.
+
For example, create a `cr_minimal.yaml` file as follows:
+
[source,options="nowrap",subs=attributes+]
----
include::cmd_examples/cat_cr_minimal.adoc[]
----
+
. Apply your `Infinispan` CR.
+
[source,options="nowrap",subs=attributes+]
----
$ {oc_apply_cr} cr_minimal.yaml
----
+
. Watch {ispn_operator} create the {brandname} pods.
+
[source,options="nowrap",subs=attributes+]
----
include::cmd_examples/oc_get_pods_w.adoc[]
----

.Next Steps

Try changing the value of `replicas:` and watching {ispn_operator} scale the cluster up or down.
